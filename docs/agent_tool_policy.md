# Agent 工具调用策略（Phase4）

## 1. 工具列表
1. `kb_retrieve`
- 用途：从团队知识库检索证据。
- 默认可执行：是。

2. `kb_ingest`
- 用途：新增文档到知识库。
- 默认可执行：否（需风控位/审批位）。

3. `kb_update`
- 用途：更新已有文档索引。
- 默认可执行：否（需风控位/审批位）。
- 当前实现约束：审批通过时需提供 `update_file_path`。

4. `web_search`
- 用途：只读外部信息补充。
- 默认可执行：是（但外部信息入库需审批）。

## 2. 风险规则
1. 仅检索 `kb_retrieve` 可直接执行。
2. `web_search` 得到结果后，若要触发 `kb_ingest/kb_update`，必须进入审批流程。
3. 未审批状态下，不允许执行知识库写入动作。
4. 所有工具调用需写入 run trace（输入摘要、输出摘要、状态、耗时）。
5. 审批通过后仅执行 `pending_action`（如 `kb_update` 或 `kb_ingest`），不强制后续固定链路。
6. 最终是否继续检索、是否重试、如何组织回答，由 LangChain Agent 自主决策。

## 3. 失败与回退
1. `kb_retrieve` 失败：返回“检索失败”，记录错误，不进行写入动作。
2. `kb_retrieve` 为空：尝试 `web_search`；若仍为空，安全拒答。
3. 低置信度：提示用户补充问题上下文或补充语料。

## 4. 权限边界
1. 所有知识库操作必须携带 `team_id`。
2. 禁止跨 `team_id` 检索与写入。
3. 高敏文档（`security_level=confidential`）需额外策略控制（后续阶段实现）。
